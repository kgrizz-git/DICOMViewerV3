# Plan: Combined PDF Report for MRI Compare-Mode Runs

**Status:** Implemented  
**Priority:** P2 — quality-of-life improvement for compare-mode workflow  
**Relates to:** ACR MRI Large compare-mode feature (implemented in WIP branch)

---

## Background and Motivation

When a compare-mode MRI analysis runs (up to 3 LC parameter sets), pylinac generates one
PDF per run via `ACRMRILarge.publish_pdf()`.  The current implementation:

- Only run 0 receives the user-chosen output path; runs 1+ skip PDF generation entirely.
- The user sees a single PDF (run 0) that includes a compact compare-mode config header in
  its notes block, but the other runs' full pylinac result pages are never saved to disk.
- The notes block on the first PDF has a hard space limit (~6 lines, ~90 chars each) imposed
  by pylinac's fixed `location=(1, 4)` placement and the "Generated by pylinac" footer at
  `y=0.5 cm`.  A proper compare summary requires far more space.

The goal of this plan is to produce a **single combined PDF** for compare-mode runs that:

1. Opens with a **viewer-generated summary page** (1–2 pages) showing the comparison table
   with all run parameters and scores.
2. Follows with **each run's full pylinac-generated PDF** concatenated in order.
3. Requires **no changes to pylinac internals**.

---

## Approach

### PDF merging library: `pypdf`

`pypdf` (successor to PyPDF2) is the recommended choice:

- Pure Python, no system dependencies.
- Can read and merge existing PDF files page-by-page.
- Actively maintained (2024–2025 releases).
- Small footprint (~500 KB wheel).

`reportlab` (already installed as a pylinac dependency) is used to generate the
viewer-authored summary pages before merging.

Add to `requirements.txt`:
```
pypdf>=4.0.0
```

### Summary page content (viewer-generated via reportlab)

**Page 1 — Compare Summary:**
- Title: "ACR MRI Large — Low-Contrast Compare Mode Report"
- Phantom details: study/series UID, date, number of images
- Run parameter table:

  | Run | Label | Method | Threshold | Sanity Mult | LC Score | Status |
  |-----|-------|--------|-----------|-------------|----------|--------|
  | 1   | Run 1 | Weber  | 0.001000  | 3.000       | 10       | OK     |
  | 2   | Run 2 | Weber  | 0.000900  | 2.700       | 10       | OK     |
  | 3   | Run 3 | Weber  | 0.001100  | 3.300       | 10       | OK     |

- MTF results from run 0 (row-wise and column-wise 50%)
- Full interpretation notes (not space-constrained as on pylinac's first page):
  - MTF meaning + docs link
  - LC score meaning (sequential stopped count, max 40)
  - Circle color meanings (blue, green, red)
  - Key parameter explanations
  - Links to pylinac docs

**Page 2 (optional) — Analysis Parameters:**
- Pylinac version, run date/time
- Full `pylinac_analysis_profile` JSON (rendered as key-value pairs)
- Viewer version

### Combined PDF assembly flow

```
run 0 → temp PDF (run0.pdf)
run 1 → temp PDF (run1.pdf)
run 2 → temp PDF (run2.pdf)   [if enabled]

viewer → summary.pdf  (1–2 pages, reportlab)

merge:  [summary.pdf] + [run0.pdf] + [run1.pdf] + [run2.pdf]
      → user-chosen output path (combined.pdf)

delete: temp PDFs
```

The user picks a single output path (the same "Save PDF" file dialog as today).
The combined PDF is written there.

## reportlab summary page design notes

Use `reportlab.platypus` (high-level layout engine) rather than raw `Canvas` calls,
so the summary page flows naturally and avoids the fixed-coordinate constraints of
pylinac's approach.

```python
from reportlab.platypus import (
    SimpleDocTemplate, Table, TableStyle, Paragraph, Spacer, PageBreak,
)
from reportlab.lib import colors
from reportlab.lib.pagesizes import A4
from reportlab.lib.styles import getSampleStyleSheet
from reportlab.lib.units import cm

def build_mri_compare_summary_pdf(batch: MRIBatchResult, output_path: Path) -> None:
    doc = SimpleDocTemplate(str(output_path), pagesize=A4,
                            leftMargin=2*cm, rightMargin=2*cm,
                            topMargin=2*cm, bottomMargin=2*cm)
    styles = getSampleStyleSheet()
    elements = []

    elements.append(Paragraph("ACR MRI Large — Low-Contrast Compare Report",
                               styles["Title"]))
    elements.append(Spacer(1, 0.5*cm))

    # Parameter and score table
    headers = ["Run", "Label", "Method", "Threshold", "Sanity ×", "LC Score", "Status"]
    rows = [headers]
    for i, (cfg, r) in enumerate(zip(batch.run_configs, batch.run_results)):
        score = str(r.metrics.get("low_contrast_score", "N/A")) if r.success else "—"
        status = "OK" if r.success else "FAILED"
        rows.append([
            str(i + 1), cfg.label, cfg.low_contrast_method,
            f"{cfg.low_contrast_visibility_threshold:.6f}",
            f"{cfg.low_contrast_visibility_sanity_multiplier:.3f}",
            score, status,
        ])
    t = Table(rows, repeatRows=1)
    t.setStyle(TableStyle([
        ("BACKGROUND", (0, 0), (-1, 0), colors.HexColor("#2c3e50")),
        ("TEXTCOLOR", (0, 0), (-1, 0), colors.white),
        ("GRID", (0, 0), (-1, -1), 0.5, colors.grey),
        ("ROWBACKGROUNDS", (0, 1), (-1, -1), [colors.white, colors.HexColor("#f2f2f2")]),
        ("FONTSIZE", (0, 0), (-1, -1), 9),
    ]))
    elements.append(t)
    elements.append(Spacer(1, 0.5*cm))

    # Full interpretation notes (no line-count constraint)
    note_style = styles["Normal"]
    note_style.fontSize = 9
    for line in _NOTES_LINES_FULL:   # longer, unwrapped version
        elements.append(Paragraph(line, note_style))

    doc.build(elements)
```

Keep `_NOTES_LINES` (the short 6-line version) for pylinac's first-page notes.
Add a separate `_NOTES_LINES_FULL` constant with the complete explanation for the
summary page.

## Edge cases to handle

| Situation | Handling |
|-----------|----------|
| Only 1 run enabled in compare mode | No compare PDF; fall back to single-run flow |
| A run fails mid-batch | Include run in table as FAILED; skip its temp PDF |
| Temp directory cleanup failure | Log warning; do not raise; combined PDF still saved |
| User cancels before batch finishes | Worker cancellation deletes temp dir |
| `pypdf` import fails (not installed) | Catch `ImportError`; fall back to single-run PDF + warning in result dialog |
