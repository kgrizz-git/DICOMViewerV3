**Location**: `README.md`  
**Type**: User-facing  
**Status**: Needs Updates (accuracy + completion in a few spots)

#### Accuracy Issues

##### Issue 1: Project structure lists non-existent directories

**Location in Documentation**: `README.md`, section "Project Structure", ~lines 173–179  
**Location in Code / repo**: Repository root (verified via glob)

**Documentation Says**: Tree includes `docs/` and `data/` under `DICOMViewerV3/`.

**Repo Actually Has**: No `docs/` or `data/` directories at project root; documentation lives in `dev-docs/`, `user-docs/`, and root markdown files.

**Impact**: Medium — new contributors or packagers may look for wrong paths.

**Suggested Fix**: Update the tree to match reality (e.g. `dev-docs/`, `user-docs/`, `scripts/`, `.github/`, etc.) or add placeholder dirs if they are planned.

##### Issue 2: Test file list is incomplete / outdated

**Location in Documentation**: `README.md`, "Running tests", ~line 329  
**Location in Code**: `tests/` (32+ `test_*.py` files)

**Documentation Says**: Tests are `test_dicom_parser.py`, `test_dicom_loader.py`, `test_dicom_utils.py`, `test_export_manager.py` only.

**Code Actually Has**: Additional modules including `test_pylinac_extent_relaxed.py`, `test_mpr_core.py`, many `tests/config/*`, `tests/metadata/*`, `tests/roi/*`, `tests/smoke/*`, etc.

**Impact**: Low–Medium — understates test coverage; misleads when searching for "all" tests.

**Suggested Fix**: Replace explicit list with "see `tests/`" or maintain a short representative list plus "and additional suites under `tests/config/`, …".

##### Issue 3: Duplicate feature bullet — Image Inversion

**Location in Documentation**: `README.md`, "Image Display", lines ~28 and ~38

**Documentation Says**: "**Image Inversion**" described twice in the same section.

**Impact**: Low — cosmetic redundancy; may confuse translation or diff churn.

**Suggested Fix**: Remove duplicate bullet.

- **Application version on landing doc**: `src/version.py` defines `__version__ = "0.1.1"`. README does not surface the current version in the header (optional but common for support requests). **Impact**: Low.
