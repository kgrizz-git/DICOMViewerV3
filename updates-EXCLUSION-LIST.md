# Exclusion List

Directories and files to exclude from automated repo scanning and processing.

## Global Exclusions

These are excluded by all update prompts:
- `.git`
- `.cursor`
- `.vscode`
- `node_modules`
- `.pytest_cache`
- `__pycache__`
- `.env`
- `.env.local`
- `.venv`

**Note:** Anything matched by `.gitignore` is also excluded automatically.

## List Generation Exclusions

Additional files/directories to exclude from `update-lists-prompt.md` (beyond global exclusions):
- `.github` (GitHub config directory — contains CI/CD workflows only, not content to catalog)

## Link Generation Exclusions

Additional files/directories to exclude from `update-links-prompt.md` (beyond global exclusions):
- `list-*.md` (autogenerated list files — do not add links to these)
- `CHANGELOG.md` (contains historical tracking, not appropriate for cross-linking)
- `VERSION` (version file, no linking needed)
- `.agent_files/` (agent-only working files — not for general cross-linking)

### Only-In Directories

Files outside can link to files within these directories, but files inside cannot link back out:
- `agents_and_skills`
- `templates-generalized`

### Only-Out Directories

Files within these directories can link to files outside, but files outside cannot link in:
