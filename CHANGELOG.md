### Fixed
- **KeyboardEventHandler import / startup:** ``__init__`` had optional parameters **before** required ones, which is invalid Python and raised **SyntaxError** on import. Parameters are reordered so required callbacks precede defaulted optional callbacks. **Semantic versioning note: patch** (startup crash).
- **ROI statistics overlay (multi-channel):** The on-image overlay previously summarized **per-channel means only**; it now also shows compact **Ch std**, **Ch min**, and **Ch max** lines when those statistics are enabled in the ROI overlay toggles (aligned with aggregate mean/std/min/max visibility). **Semantic versioning note: patch** (UX parity with the statistics panel / export).
- **Cine video export (0.2.10):** **MP4** export (**MPEG-4 Part 2**, ``+faststart``) for **Windows 11 Media Player**, which documents in-box **``.mp4``** playback for MPEG-4 but treats **``.mpg``** as **MPEG-1/2** (often **no playback and no codec prompt** unless the free **MPEG-2 Video Extension** is installed ‚Äî putting MPEG-4 video in ``.mpg`` did not help). **MPG** again uses standard **MPEG-2** in a program stream for extension-equipped / VLC workflows. **Export Cine** dialog defaults to **MP4**; filters and docs updated. Tests: **`tests/test_cine_video_export.py`**. **Semantic versioning note: patch** (export / playback compatibility).
- **Cine video export (0.2.9):** **GIF** frame timing ‚Äî imageio‚Äôs **Pillow** GIF writer expects ``duration`` in **milliseconds**; the app previously passed **seconds**, so delays were effectively zero and many viewers clamped every GIF to the same speed. *(An interim **MPG** = MPEG-4-in-program-stream experiment for WMP was superseded in **0.2.10** by **MP4** + restored **MPEG-2** ``.mpg``.)* Tests: **`tests/test_cine_video_export.py`**. **Semantic versioning note: patch** (export bug fix / playback compatibility).
- **Cine video export (0.2.8):** **AVI** no longer uses PNG-in-AVI (**MPNG** in Windows Media Player); encoding uses **MPEG-4 Part 2** (`mpeg4`) with **YUV 4:2:0**. **Raster parity with PNG/JPG export:** cine dialog adds **resolution** (magnification) options; **`main.py`** passes **`export_scale`**, annotation aggregation (**`subwindow_annotation_managers`**), and **`scale_annotations_with_image=False`** like **Export Images**. UI strings use title case (**Export Cine As‚Ä¶**). Tests: **`tests/test_cine_video_export.py`**. **H.264/MP4 not added** (patent/redistribution policy left for a separate product decision; **AGENTS.md** FFmpeg **LGPL/GPL** note still applies). *(Initial **MPG** / **GIF** timing choices were revised in **0.2.9**.)* **Semantic versioning note: patch** (export bug fix / compatibility).
