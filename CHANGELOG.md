# Changelog

All notable changes to DICOM Viewer V3 are documented here. The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the project follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html). See [dev-docs/info/SEMANTIC_VERSIONING_GUIDE.md](dev-docs/info/SEMANTIC_VERSIONING_GUIDE.md) for version increment rules.

## [Unreleased]

### Fixed
- **Annotation paste handler startup**: `src/core/annotation_paste_handler.py` imports `from __future__ import annotations` so return types like `list[ROIItem]` are not evaluated at class definition time. Previously `ROIItem` lived only under `TYPE_CHECKING`, which caused `NameError` when starting the app.
- **ROI manager Pyright/Based Pyright errors**: `src/tools/roi_manager.py` now uses a `RoiStatisticsMap` type for ROI statistics (float / int counts / optional `area_mm2`), types statistics overlays as `DraggableStatisticsOverlay`, allows `roi` on the overlay to be cleared when disconnecting, annotates `ConfigManager | None` and `QGraphicsScene` parameters, and resolves overlay font-size narrowing after config reads. Adds `clear_deleted_flag` / `set_updating_position` helpers on the draggable overlay.
- **ROIItem move callback typing**: `ROIItem.on_moved_callback` is explicitly typed as `Optional[Callable[[], None]]` so assignments from coordinators/paste logic are valid for the type checker; previously it was inferred as `None` only (`src/tools/roi_manager.py`).
- **Arrow annotation selection**: Selected arrows use a tight dashed yellow outline drawn from the visible shaft and mapped arrowhead path instead of Qt’s default selection frame (avoids Qt’s default selection box). When selected, `boundingRect()` unions the dashed-outline hull with pen margin and `prepareGeometryChange()` runs on selection, moves, and endpoint updates so Graphics View repaints the full stroke region, eliminating yellow dashed outline pixel trails while dragging (`src/tools/arrow_annotation_tool.py`).
- **Pyright type warnings**: Resolved Pyright typing diagnostics in `src/main.py` by tightening Qt/viewer typing, optional narrowing, and callback/slot contracts.
- **dicom_loader pyright type errors**: Normalized `Optional[str]` validation error messages to `str` when recording failed DICOM file paths in `src/core/dicom_loader.py`, aligning with `failed_files: List[Tuple[str, str]]` and eliminating Based Pyright argument-type issues.
- **dicom_organizer pyright type errors**: Avoided passing `List[Optional[str]]` file paths into `_organize_files_into_batch` by constructing `Optional[List[str]]` in `src/core/dicom_organizer.py`, eliminating Based Pyright `reportArgumentType` diagnostics.
- **presentation_state_handler pyright type errors**: Normalized graphic type parsing and coordinate tuple/float coercion so `src/core/presentation_state_handler.py` matches its `basedpyright` expectations.
- **ROI/export/slice/tag pyright type errors**: Coerced optional numeric stats and added `None` guards for export + geometry + workbook sheet removal in `src/core/roi_export_service.py`, `src/core/slice_geometry.py`, and `src/core/tag_export_writer.py`.
- **Slice sync/display + tag edit/view state pyright type errors**: Parameterized `Callable`/`dict`/`list` generics and narrowed tag edit command typing in `src/core/slice_display_manager.py`, `src/core/slice_location_line_coordinator.py`, `src/core/slice_sync_coordinator.py`, `src/core/subwindow_lifecycle_controller.py`, `src/core/tag_edit_history.py`, and `src/core/view_state_manager.py`.
- **MPR modules basedpyright errors**: Aligned `src/core/mpr_builder.py`, `src/core/mpr_volume.py`, and `src/core/mpr_cache.py` with the project’s SimpleITK typing pattern (`sitk` as `Any`, `sitk_available` flag), replaced invalid `sitk.Image` type forms with `Any` where needed, parameterized `dict`/`tuple` generics, and removed the `TYPE_CHECKING` import of `main.DICOMViewerApp` from `src/core/mpr_controller.py` (constructor takes `app: Any`) to eliminate `reportImportCycles` with `src/main.py`.
- **Export + loading pipeline + SimpleITK basedpyright errors**: Fixed an ROI/measurement drawing indentation bug in `src/core/export_manager.py` (preventing unbound ROI/measurement variables), tightened callback/Qt typing in `src/core/file_operations_handler.py`, `src/core/loading_pipeline.py`, and `src/core/loading_progress_manager.py`, and cleaned up SimpleITK typing + index/float coercions in `src/core/image_resampler.py`. Also normalized resampling cache key typing and missing `Dict[...]` generics in `src/core/fusion_handler.py` and `src/core/file_series_loading_coordinator.py`.
- **pydicom `convert_color_space` import**: Updated import path to `pydicom.pixels.processing` for newer `pydicom` exports in `src/core/dicom_color.py` (and `src/core/dicom_processor.py`).
- **dicom_color Pyright constant names**: Renamed module-level `PYDICOM_CONVERT_AVAILABLE` to `pydicom_convert_available` (Based Pyright treats all-caps as constants). YBR→RGB 2D helper uses `r_ch` / `g_ch` / `b_ch` instead of uppercase `R` / `G` / `B` to avoid `reportConstantRedefinition` across branches (`src/core/dicom_color.py`).

### Changed
- **MPR “Combine Slices” at display time**: `MprBuilder` now always stores the **uncombined** MPR stack. Slab projection (AIP/MIP/MinIP and slice count) runs in `MprController.display_mpr_slice` from per-subwindow `mpr_combine_*` state. The right-pane **Combine Slices** widget updates that state and redisplays the MPR (same as for axial series projections). Disk cache format is bumped to **v4** (cache keys no longer include slab parameters), so prior v3 entries are not reused. Create-MPR dialog slab choices still apply as the initial `mpr_combine_*` seed when MPR is activated.
- **QI assessment template**: `dev-docs/templates-generalized/qi-assessment-template.md` now includes Pyright/Based Pyright steps (`src` + `tests`), `requirements-dev.txt` install, venv `Scripts`/`bin` invocation, log capture to `dev-docs/code-assessments/pyright-<timestamp>.txt`, and a ≤10-line summary section for timestamped assessments.
- **`requirements-dev.txt`**: Optional **`basedpyright`** dependency so `basedpyright` is available on PATH after `pip install -r requirements-dev.txt` (CLI type-check aligned with Pyright/Based Pyright in the IDE).
- **Docs**: `AGENTS.md` notes a ~10 minute ceiling for long-running automation commands (e.g. full-tree typecheck or full test suite) when appropriate.
- **Fusion debug logging**: Console prints from `fusion_coordinator.get_fused_image`, overlay series auto window/level setup, and `fusion_processor.create_fusion_image` (spacing, scaling, translation, normalized overlay stats) are gated behind `DEBUG_OFFSET` in `utils/debug_flags.py` (default `False`), consistent with `fusion_controls_widget` offset tracing.
- **Docs**: `AGENTS.md` and `README.md` now document virtual environments under `venv`, `.venv`, and other common folder names, consistent with `launch.bat` resolution order.
- **Pyright venv config**: `pyrightconfig.json` now points to `.venv` (instead of `venv`) so local type-check runs use the intended environment.
- **launch.bat**: When no virtual environment is found, the launcher briefly explains what a venv is, why it helps, that using one is recommended but optional, and that system Python remains available. Also resolves an existing environment under the project root in order: `venv`, `.venv`, `env`, `virtualenv` (each must contain `Scripts\activate.bat`). Create/setup still targets `venv` by default when missing.
- **Docs**: `requirements.txt` and `README.md` note that Windows installs work best on Python **3.11/3.12** (pre-built wheels for `pyjpegls`); **3.14** may require MSVC Build Tools to compile `pyjpegls`, and Parallels + `\\Mac\Home\...` paths can complicate venv/pip.
- **Security tooling docs/setup**: `dev-docs/SECURITY_TOOLS_CLI_GUIDE.md`, `README.md`, and `AGENTS.md` now treat TruffleHog **v3 binary install** as the recommended local path (matching CI) and keep `requirements-dev.txt` focused on Python-installable scanners (`semgrep`, `detect-secrets`).

- **Planning docs**: `dev-docs/plans/HANGING_PROTOCOLS_PRIORS_RDSR_PLAN.md` outlines hanging protocols, prior-study loading after a local database exists, and RDSR (radiation dose SR) parsing/export with in-repo sample data; `dev-docs/TO_DO.md` links those checklist items to the plan.
- **Planning docs**: `dev-docs/plans/MPR_DICOM_SAVE_CINE_VIDEO_EXPORT_ANGLE_MEASUREMENT_PLAN.md` outlines implementation for saving MPR stacks as DICOM, cine video export (MPG/GIF/AVI), and an angle measurement tool; `dev-docs/TO_DO.md` links those checklist items to the plan.
- **Planning docs**: `dev-docs/plans/MPR_ANNOTATIONS_AND_RIGHT_PANE_COMBINE_SLICES_PLAN.md` outlines text/arrow annotation support on MPR views and driving MPR slab combine from the right-panel “Combine Slices” control (not only from the Create MPR dialog); `dev-docs/TO_DO.md` links the related checklist items.
- **MPR slab combine + ROI/measurement tools**: The MPR creation dialog supports slab combine modes (`None`, `MIP`, `MinIP`, `AIP`) with configurable slab thickness, and MPR subwindows now allow ROI drawing + distance measurements with ROI statistics driven by the actual MprBuilder output pixels.
- **`requirements-dev.txt`**: Optional pip file that includes `requirements.txt` plus local Python security scanners (**semgrep**, **detect-secrets**) documented in `dev-docs/SECURITY_TOOLS_CLI_GUIDE.md`. Use `pip install -r requirements-dev.txt` for contributor / pre-commit-style scanning; end users can keep using `requirements.txt` only.
- **`scripts/install-trufflehog-v3.ps1`**: Helper script to install official TruffleHog v3 Windows binaries from GitHub releases, with optional user-PATH update and version pin support.
- **Slice location lines across views**: Optional visualization showing the intersection of other views' slice planes on the current image. Toggle in View → Show Lines (Enable/Disable, Only Show For Same Group, Show Only For Focused Window) and in the image context menu (Show Lines submenu). Line colors match subwindow dot colors (blue, green, orange, magenta). Works with or without slice sync; when "Only Show For Same Group" is enabled, only subwindows in the same linked group show their slice line; when "Show Only For Focused Window" is enabled, only the focused subwindow's line is shown. Requires DICOM geometry (ImagePositionPatient, ImageOrientationPatient).
- **Single-slice MPR**: MPR can be built from a series with a single slice (or a single unique position after deduplication), producing a "short" one-slice volume; the MPR dialog and volume builder now allow at least one slice instead of requiring two.
- **MPR SliceLocation fallback**: When a series has no ImagePositionPatient but SliceLocation (0018,0050) is populated, the user is asked whether to use SliceLocation for slice order and MPR. If they accept, slice position is derived from SliceLocation along the slice normal so MPR can still be built. The MPR dialog is enabled for such series (availability check considers SliceLocation when position is missing).
- **MPR mixed-orientation choice**: When building MPR from a series with slices in more than one orientation, a dialog lists each orientation (Axial, Coronal, Sagittal, or Oblique) with the image count (e.g. "Axial — 42 images"). The user selects one; MPR is built from that group only.
- **Slice sync and MPR groundwork**: Added patient-space slice geometry primitives, optional linked-group slice synchronisation, MPR volume/build/cache modules, an MPR creation dialog, and initial subwindow wiring for synthetic MPR views with on-image `MPR` banner rendering and MPR-aware slice navigation.
- **Slice Sync in image context menu**: The image viewer right-click menu now includes a `Slice Sync` submenu with `Enable Slice Sync` and `Manage Sync Groups...`, matching the existing `View` menu controls for linked-window scrolling.
- **Trackpad pinch-to-zoom**: Pinch on a trackpad zooms the image in/out without changing scroll-wheel behavior. On macOS and Wayland, native pinch gestures are handled via `QNativeGestureEvent` (ZoomNativeGesture). On Windows, Ctrl+wheel is treated as zoom so trackpad pinch (when the OS sends it as Ctrl+wheel) also zooms. Scroll wheel mode (Slice vs Zoom) continues to apply only to the physical wheel; pinch always zooms.
- **Distance measurement positioning UX**: When a measurement is selected, small yellow cross markers show at both line endpoints so ends stay visible. While dragging a measurement handle, handles are hidden so the endpoint and image are unobstructed; handles reappear on release. A magnifier appears by default during handle drag for precise placement; the magnifier hides when the drag ends. Cursor is hidden during initial measurement (after first click until second click) and during measurement handle drag. See `dev-docs/plans/distance-measurement-positioning-ux-plan.md`.
- **Quick Window/Level (Q)**: Context menu item "Quick Window/Level (Q)" and keyboard shortcut **Q** open a small dialog to enter window center and width (Tab between fields, Enter or OK to apply). Applies to the focused subwindow; values and ranges match the right-panel WL controls (rescaled or raw).
- **Window/level debug logging**: `DEBUG_WL` in `utils/debug_flags.py` (default `False`). Set to `True` to trace default WL when switching series: at each display_slice it logs study/series, modality, is_new_study_series, is_same_series, rescale slope/intercept, use_rescaled_values; whether the series was found in current_studies; embedded WL source (first preset vs get_window_level_from_dataset) and raw wc/ww/is_rescaled; any rescaled↔raw conversion; stored_wc/stored_ww; fallback path (when series not in current_studies) with same detail; and final window_center/window_width/apply_rescale passed to dataset_to_image. Use when investigating embedded WL not applied correctly across studies.
- **Series navigation debug logging**: All `[DEBUG-NAV]` and "Series navigation:" print statements are gated behind a new `DEBUG_NAV` flag in `utils/debug_flags.py`. Default is `False`; set to `True` to restore verbose series-navigation key/request/lock tracing in `series_navigator`, `image_viewer`, and `file_series_loading_coordinator`.
- **Status bar (processed vs loaded)**: After loading, the status bar shows "X files processed - Y studies, Z series, A files loaded from {source}" with optional " (B non-DICOM and C duplicate files not loaded)" when applicable, so counts stay accurate when canceling, when some files are non-DICOM, or when duplicates are skipped.

They are excluded from "files processed" and from failed/duplicate counts. Open Files and drag-and-drop file lists are filtered before loading; folder load excludes them during directory scan. If all selected files are skipped, a short warning is shown.

After additive load (Open Files, Open Folder, Recents, drag-and-drop), status bar shows: "Loaded {n} new series across {m} studies", "Added {k} slice(s) to existing series", or "No new files — all {total} already loaded". When any files were skipped as duplicates, a toast message "{n} file(s) already loaded and skipped" appears at bottom-center and auto-dismisses after 3 seconds (implemented via `MainWindow.show_toast_message()`).
- **Export (anonymized DICOM)**: When "Anonymize patient information" is enabled, the export folder structure (Patient ID / Study Date - Study Description / Series Number - Series Description) is now built from anonymized tag values so folder paths do not leak patient data.
- **SliceDisplayManager.clear_display_state()**: Clears current_dataset, current_studies, and UIDs so no stale cached state is used after close/open.

State is persisted via existing `splitter_sizes` config. View menu check state stays in sync when the user drags the splitter.

### Changed
- **Show Lines menu**: Slice location lines are now under View → Show Lines (and image context menu → Show Lines) as a separate submenu, not under Slice Sync. The submenu has "Enable/Disable" (checkmark when enabled), "Only Show For Same Group" (checkmark when true), and "Show Only For Focused Window" (checkmark when true). When "Show Only For Focused Window" is enabled, only the focused subwindow's slice plane intersection is drawn on other views; lines update when focus changes.
- **Slice location line colors**: Line colors now match the subwindow dot colors (blue, green, orange, magenta) from the series navigator, so each window's line is visually associated with its colored dot.
- **Show Lines at launch**: When slice location lines are enabled in config from a previous session, they are now applied correctly when files are loaded at application launch (initial load and additive load).
- **Show Lines on load/assign**: Slice location lines now appear immediately when loading new files or assigning a series to a subwindow, without requiring a slice scroll first. A deferred refresh (100 ms) ensures the display has settled before drawing lines.
- **Slice sync groups on close/exit**: When File → Close All is used or when the application exits, any existing slice sync groups are dissolved (cleared and persisted) so the next session or after reopening files starts with no linked groups.
- **Slice location lines default**: `slice_location_lines_same_group_only` now defaults to `False` so all views show lines from all other views. Previously, with default `True` and a linked group like [0,2], views 0 and 2 only showed each other's lines while views 1 and 3 showed all—causing inconsistent behavior. Enable "same group only" in settings for linked-group-only behavior.
- **Handle-drag magnifier by default**: The magnifier now appears automatically when dragging a measurement handle (no Shift required). Previously it only appeared with Shift+drag.
- **Magnifier shows annotations**: The cursor magnifier and the handle-drag magnifier now use scene rendering instead of image-only extraction, so measurements, ROIs, and other overlay items are visible in the magnified view.
- **Series navigator**: Series number labels (e.g. "S01") in the navigator thumbnails use a smaller font (9 pt instead of 12 pt).
- **Status bar**: Removed the "X files processed - " prefix. Status line is now "{studies}, {series}, {files} loaded from {source}". When any files were skipped, the suffix is "(Y non-DICOM, Z duplicates skipped)" (or only one of Y/Z when applicable); Y combines extension-skipped and failed-to-load non-DICOM counts.
- **Privacy**: Crosshair managers are no longer updated when toggling privacy mode; crosshairs always show full content and do not hide anything in privacy mode, so the call was unnecessary and could crash when Qt objects were already deleted.
- **Clear/close**: When closing files or opening new folder/files, all subwindows' `slice_display_manager` state (current_dataset, current_studies, UIDs, slice index) is now cleared via `clear_display_state()`, preventing stale redisplay (e.g. series reappearing in another window on privacy toggle).
- **Refactor (main.py)**: Privacy propagation and overlay refresh after privacy change moved to `src/core/privacy_controller.py`. `_on_privacy_view_toggled` and `_refresh_overlays_after_privacy_change` now delegate to `PrivacyController`; behavior unchanged.
- **Refactor (main.py)**: Main-window panel layout assembly moved to `src/gui/main_window_layout_helper.py`. `_setup_ui` now delegates to `setup_main_window_content()`; behavior unchanged (center/left/right panels, tabs, series navigator, window-slot map callbacks).
- **Refactor (main.py)**: Customization and tag-preset export/import logic moved to `src/core/customization_handlers.py`. `_on_export_customizations`, `_on_import_customizations`, `_on_export_tag_presets`, and `_on_import_tag_presets` now delegate to a `CustomizationHandlers` helper; post-import apply logic remains in main via `_apply_imported_customizations` callback. Behavior unchanged.

### Fixed
- **MPR clear + overlay anchoring**: `Clear MPR View` now actively clears a subwindow when the MPR replaced an otherwise empty pane, instead of leaving the last reconstructed slice visible. MPR slices now also update the normal view-state context so metadata overlays and the `MPR` banner stay viewport-anchored during pan/zoom like regular series overlays.
- **MPR + series assignment**: Left/right series navigation, navigator thumbnail clicks, and context-menu "Assign Series to Focused Window" are blocked when the focused subwindow is in MPR mode. A non-blocking toast is shown: "Please clear MPR before assigning a new series to Window X."
- **MPR mixed orientations**: Building an MPR volume now checks slice orientations (ImageOrientationPatient). If the series contains slices in more than one orientation, a dialog asks the user which orientation to use and shows how many images are in each group (e.g. "Axial — 42 images", "Coronal — 10 images"). MPR is built from the selected group only. Single-orientation series builds as before.
- **File → Close All + MPR**: Close All now clears MPR from all subwindows and removes the MPR banner before clearing overlays and data.
- **MPR Create dialog + orientation choice**: The Create MPR dialog now closes as soon as OK is clicked, before the orientation-choice or build logic runs. This prevents the dialog from staying open and blocking the orientation-choice or error message dialogs when building from a series with multiple orientations.
- **MPR coronal/sagittal black output**: Corrected the SimpleITK direction-matrix axis mapping and the MPR output-grid row/column origin mapping so orthogonal reconstructions no longer collapse into empty or thin black-bar slices. This is a backward-compatible bug fix and does not change the planned minor-version impact of the unreleased slice-sync/MPR feature work.
- **Layout map popup UnboundLocalError**: Opening the window-slot map from the context menu (Swap → Show Window Map) no longer raises "cannot access local variable 'widget'" on first open; the popup widget reference is set in both code paths.
- **Histogram open and W/L responsiveness**: When the histogram dialog is open, window/level slider and slice changes no longer block the UI. W/L changes use a light update (no pixel refetch, 100 ms throttle) so the sliders stay responsive; slice changes use a full update (300 ms throttle). The histogram widget uses `draw_idle()` so matplotlib redraws are deferred and the main thread stays responsive.
- **Measurement line jitter and displacement during handle drag**: During handle drag, `update_distance()` was still calling `setPos(self.start_point)` and `update_handle_positions(force=True)`, causing redundant updates and jitter; the line could also stay displaced after release. `update_distance()` now skips those steps when `_handle_drag_in_progress` is True (the handle's itemChange already sets group position and the other handle). On drag end, `show_handles_after_drag()` now calls `update_distance()` once so line/text/group are in sync before repositioning handles.
- **Measurement line jump during handle drag**: While dragging a measurement handle, the group's position-sync logic could run after the handle's itemChange returned (when `_updating_handles` was already False), applying an incorrect delta and causing the line to jump. The group's `ItemPositionChange` and `ItemPositionHasChanged` handlers now also skip when `_handle_drag_in_progress` is True, so the line stays stable for the full duration of the drag. `MeasurementItem` also now sets `ItemSendsGeometryChanges` so Qt's internal `movingItems` mechanism cannot silently move the group while a handle is being dragged.
- **Endpoint crosshair shown on dragged handle**: When dragging a measurement handle while the measurement is selected, the yellow cross marker at the dragged endpoint was still painted, obscuring the exact tip position. `MeasurementItem.paint()` now skips drawing the crosshair for whichever endpoint corresponds to the handle currently being dragged.
- **Handle-drag magnifier (Shift+drag) not appearing**: `ImageViewer.show_handle_drag_magnifier` and `update_handle_drag_magnifier` derived the global widget position via `mapFromScene` → `mapToGlobal`, which silently raised an exception inside PySide6's C++ event dispatch during a drag, suppressing all code after it (including the `show_at_position` call). Both methods now use `QCursor.pos()` to obtain the global position directly, matching the working approach used by the main magnifier tool.
- **Default window/level overwritten after series switch**: When switching from a series with large WL (e.g. PET 35121/63217) to one with small embedded WL (e.g. CT 30/100), the WL controls were updated after `set_ranges()`. Changing the spinbox range caused Qt to clamp the old values and emit `valueChanged`, so `handle_window_changed` overwrote view state with the clamped values (e.g. 235/63217). Subsequent `display_slice` (same series) then read those wrong values. The fix is to set the WL control values (and unit) **before** calling `set_ranges()` for a new series, so the spinboxes already hold the correct values and no clamp/emit occurs.
- **Default window/level when switching series across studies**: When switching to a series from a different study, if the series lookup in `current_studies` did not yield window/level (e.g. key mismatch or missing entry), the image could be built without applying the new series’ embedded WL and the WL control ranges could stay from the previous series. A fallback now computes embedded window/level from the dataset being loaded (presets or single tags, with correct rescaled/raw conversion), sets the series pixel range from that slice, and applies the values so both the image and the right-panel controls use the correct default for the new series. Additionally, the image conversion now always receives explicit window_center/window_width whenever we have them (not only when is_same_series), so the image and controls use the same values and rescale state.
- **First series auto-loaded**: The series auto-displayed when opening a batch is chosen using the same logic as the series navigator: first study in navigator order (dict iteration), then series with lowest SeriesNumber in that study. So the auto-loaded series is always from the first study shown in the navigator; both additive and replace load paths use this logic.
- **Toast**: Toast message now stays visible longer (default 5 s), uses a larger font (14px) and padding, and is positioned higher (100px from bottom) for better visibility.
- **Fit-to-view on cancel/partial load**: When loading is cancelled but some files were already loaded, a deferred fit-to-view is now applied to the subwindow that received the loaded series so the image is correctly fitted after layout is stable.
- **Fit-to-view for new series**: Additive load schedules a deferred fit-to-view on the target subwindow's viewer so fit is applied reliably for newly loaded series (including when the new series goes to a non-focused subwindow).
- **Window/level wrong subwindow**: When a new series is added to a non-focused subwindow, the global window/level controls and metadata panel are no longer updated with the new series' values; only the subwindow that received the new series is updated, so the focused subwindow's W/L is not overwritten.
- **Status bar**: Corrected typo "studyies" to "study" / "studies" in the final load status message. Status bar now shows only the current batch’s study/series/file counts (from `MergeResult`), so counts are accurate when loading is cancelled (partial load) or when duplicate files are skipped.
- **Progress dialog on cancel**: When loading is cancelled before all files are loaded, the progress dialog and status bar now show the number of files actually loaded (e.g. 6) instead of the batch total (e.g. 11). Loader final progress uses `len(loaded_files)`; handler uses the callback’s current (actual) count for the "Loaded X file(s). Organizing..." message.
- **Privacy + crosshair**: Toggling privacy after drawing a crosshair and then loading a different exam no longer raises `RuntimeError: Internal C++ object (DraggableCrosshairText) already deleted`. CrosshairItem.update_privacy_mode now catches RuntimeError when the Qt text item was already deleted; PrivacyController also catches RuntimeError when calling crosshair_manager.set_privacy_mode.
- **Privacy + reload study**: Toggling privacy after loading a different study and then reloading the first study no longer redisplays stale series in non-focused windows. PrivacyController.refresh_overlays now only refreshes subwindows that have an entry in subwindow_data (i.e. have loaded data), so windows that were not updated when the other study was loaded are skipped.
- **GitHub CI**: Removed Dependabot `docker` updates (no root `Dockerfile`). Semgrep uses `pip install semgrep` and `semgrep scan --sarif-output semgrep.sarif` (legacy `semgrep-action` did not support `generateSarif`).
